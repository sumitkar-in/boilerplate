# Local Setup Guide

This guide covers everything you need to get the boilerplate running on your local machine for development.

## 1. Prerequisites

Before starting, ensure you have the following installed on your system:
- **Node.js**: v20 or newer
- **pnpm**: v8 or newer (Install via `npm install -g pnpm`)
- **Docker & Docker Compose**: Required for running the database, Redis, and observability stack. (e.g., Docker Desktop)

## 2. Environment Configuration

1. Copy the example environment file to create your local `.env`:
   ```bash
   cp .env.example .env
   ```
2. Open `.env` and review the variables. For a standard local setup, the default values are sufficient. The system is pre-configured to use the Docker-hosted Postgres and Redis instances.

## 3. Start Infrastructure (Docker)

Start the underlying services (PostgreSQL, Redis, PgAdmin, Grafana, etc.) using Docker.

```bash
pnpm docker:up
```

*Note: This command automatically generates local SSL certificates using `mkcert` (or falls back to OpenSSL) and brings up the containers in detached mode.*

## 4. Install Dependencies

Install the project dependencies using pnpm:

```bash
pnpm install
```

## 5. Database Setup & Seeding

Once the Docker containers are healthy and dependencies are installed, you need to migrate and seed your local database.

Run the fresh database command to clean any existing state, run core migrations, tenant migrations, and seed the super admin user:

```bash
pnpm db:fresh
```

**Important:** Keep an eye on the terminal output during this step. The seed script will output a securely generated **temporary password** for the `admin@example.com` super admin account. You will need this password to log in.

## 6. Development Modes

The boilerplate supports different development modes depending on your needs. Choose the mode that fits your current task.

### Mode A: Standard Host Development (Recommended)

In this mode, the API and Web applications run directly on your host machine for the fastest hot-reloading and easiest debugging, while only the database and Redis run in Docker.

1. In `.env`, ensure `COMPOSE_PROFILES=minio` (or empty if not using object storage).
2. Start the infrastructure:
   ```bash
   pnpm docker:up
   ```
3. Run the development servers in separate terminal tabs:
   ```bash
   pnpm dev:api      # Terminal 1: NestJS API (runs on port 3000)
   pnpm dev:worker   # Terminal 2: Background job processor
   pnpm dev:web      # Terminal 3: React Vite app (runs on port 5173)
   ```

### Mode B: Dockerized API & Nginx Mode

In this mode, the API and Nginx reverse proxy run entirely inside Docker. This closely mimics production routing (routing both `/api` and `/` through Nginx on port 443).

1. In `.env`, update your compose profiles:
   ```bash
   COMPOSE_PROFILES=minio,api
   ```
2. Start the infrastructure (this will now build and boot the API and Nginx containers):
   ```bash
   pnpm docker:up
   ```
3. Run only the frontend development servers on your host machine:
   ```bash
   pnpm dev:web      # Terminal 1: React Vite app
   ```
*(Note: Do NOT run `pnpm dev:api` in this mode, as both the host and Docker will fight over port 3000).*

### Mode C: Full Observability Mode

If you are working on performance, debugging metrics, or viewing structured logs in Loki, you can boot the entire observability stack alongside your environment.

1. In `.env`, enable observability and update your profiles:
   ```bash
   COMPOSE_PROFILES=observability,minio
   OBSERVABILITY_ENABLED=true
   ```
2. Start the infrastructure:
   ```bash
   pnpm docker:up
   ```
3. Boot your host development servers as usual:
   ```bash
   pnpm dev:api
   pnpm dev:worker
   pnpm dev:web
   ```

## 7. Accessing the Application

With everything running, you can access the various parts of the stack:

- **Web App**: [https://localhost](https://localhost)
- **API Base URL**: [https://localhost/api](https://localhost/api)
- **PgAdmin (Database GUI)**: [http://localhost:5050](http://localhost:5050)
- **Grafana (Observability)**: [https://localhost/grafana](https://localhost/grafana)

### Super Admin Login

To access the platform admin dashboard:
1. Navigate to [https://localhost/super-admin/login](https://localhost/super-admin/login)
2. Log in with:
   - **Email**: `admin@example.com`
   - **Password**: *(The temporary password generated by `pnpm db:fresh`)*

## Need to Wipe Everything?

If you ever need to completely reset your local environment and wipe all data volumes:

```bash
pnpm docker:down
docker compose -f infra/docker/docker-compose.yml --env-file .env down -v
pnpm docker:up
pnpm db:fresh
```
